# Catnip Forge 软件使用手册

> 本文是内置软件助手“猫薄荷”的产品知识来源。修改并保存本文后，下一次提问立即使用新内容，无需重启软件。

## 1. 产品定位

Catnip Forge 中文全称为“Catnip 硬件智能开发平台”，英文定位为“Autonomous Hardware Development Agent”。它是一款面向硬件开发的本地桌面 IDE，主要用于 ESP32/ESP32-S3 工程创建与修改、编译、烧录、串口调试、代码编辑和 Agent 辅助开发。

当前发布版为 v1.0.0，内部构建号为 7201。

## 2. 第一次启动与 API Key

1. 完整解压 Catnip Forge 便携目录，不能只复制 `Catnip Forge.exe`。
2. 建议放在路径较短的普通可写目录，例如 `D:\CatnipForge`，不要直接放入需要管理员权限的目录。
3. 第一次打开时，软件会要求输入 DeepSeek API Key。
4. 保存成功后，Key 写入当前软件目录的 `resources\apikey.txt`，软件会自动重启一次。
5. 不要向其他人发送包含自己 `apikey.txt` 的软件目录。

如果保存后 Agent 不可用，应先确认网络可访问 DeepSeek、Key 有效，并检查软件所在目录是否允许写入。

## 3. 主界面布局

- 左侧是 Agent 对话区，用于描述硬件开发任务和查看 Agent 回复。
- 右侧包含“仓库”“监视器”“任务管理器”和“编辑器”四个主要标签页。
- 右下角的猫咪形象是软件使用助手“猫薄荷”，用于回答界面和操作问题，不直接执行硬件任务。

## 4. Agent 对话与历史会话

- 在左侧输入框中描述任务，发送后 Agent 会分析并执行。
- Agent 工作期间，“执行过程”卡片显示当前阶段；详细工具日志默认收起，专业用户可展开查看。
- 左侧“历史对话”可以新建和切换会话，软件关闭并重新打开后仍会恢复。
- 每条历史会话右侧的“⋯”菜单支持重命名、置顶、取消置顶和删除。
- Agent 正在工作时，会话切换和部分编辑操作会暂时锁定，避免结果写入错误会话。
- 需要编译、烧录、创建工程、修改代码或操作文件时，应在这里向 Agent 提出要求，而不是让“猫薄荷”执行。

## 5. Skills 使用和维护

### 在对话中使用 Skill

1. 点击左侧 Agent 输入区附近的 `Skills` 按钮。
2. 从弹出的列表中选择需要的 Skill。
3. 被选中的 Skill 会作为标签进入输入区。
4. 输入具体任务并发送，软件会自动把对应 Skill 交给 Agent，无需手工记忆 `/skill-id`。

### 管理 Skill

- 打开右侧“仓库”标签页，在 Skills 区域可以查看、新建、编辑、删除和同步 Skill。
- `立即同步` 会把源仓库中的 Skill 部署到 Agent 工作区。
- 发布版的用户可维护源目录是 `resources\agent\skills`。
- 新增标准 Skill 时，推荐使用 `<skill-id>\SKILL.md` 目录结构，并在文档顶部提供 `name` 和 `description`。

## 6. 仓库标签页

“仓库”用于查看和管理 Catnip Forge 可访问的资源：

- 硬件工程：可编译或烧录的工程源码、CMake 文件和配置文件。
- 参考代码：ESP-IDF 示例和可复用代码片段。
- Skills：Agent 的专业工作流和操作说明。

卡片上的“在资源管理器中打开”会打开对应目录。编辑源文件时可使用文件条目旁的编辑入口跳转到内置编辑器。

## 7. 任务管理器：编译与烧录

### 编译

1. 打开“任务管理器”。
2. 点击“刷新工程”，然后选择工程。
3. 点击“编译”。
4. 状态胶囊、进度区域和下方任务结果会显示编译状态与日志。

### 烧录

1. 先用“刷新设备”重新枚举串口。
2. 选择正确的 COM 端口。
3. 点击“烧录”。
4. 如果提示端口被占用，应先关闭“监视器”中的串口，或关闭其他串口助手，再重试。

“清除记录”会清除任务历史和相关运行日志，但不会删除硬件工程源码。

## 8. 监视器：串口收发

“监视器”是内置双向串口助手，支持：

- 选择串口和刷新设备。
- 设置波特率、数据位、停止位和校验位。
- 文本或 HEX 模式收发。
- GBK、UTF-8、ASCII 和 Latin1 文本编码。
- 配置发送行尾。

常见 ESP32 日志波特率是 115200，但应以具体工程配置为准。一个串口同一时间通常只能被一个程序打开；烧录、串口监视器或外部串口工具相互冲突时，会出现“拒绝访问”或端口占用错误。

## 9. 编辑器

- “编辑器”支持从受控资源目录打开和修改代码、CMake、Markdown、JSON 等文件。
- 左侧资源树用于切换目录和文件，顶部标签用于切换已经打开的文件。
- 修改后应执行保存，再进行编译。
- 字号工具提供减小、当前字号和增大按钮，字号会持久化。
- 未打开文件时，编辑区会显示空状态提示。

## 10. 猫薄荷软件助手与外观

- 点击右下角猫咪形象会打开微型聊天框。
- 猫薄荷只回答 Catnip Forge 的使用、界面和故障排查问题，不具备文件修改、编译、烧录或删除权限。
- 聊天框顶部可切换深色或浅色主题。
- `−` 和 `＋` 可以调整猫咪形象大小。
- 猫咪可以拖动到其他位置，位置和大小会自动保存。
- 回答由 DeepSeek 生成；遇到手册没有说明的功能，猫薄荷应明确表示不确定，不能编造菜单或按钮。

## 11. 常见问题

### 串口列表为空

点击“刷新设备”，确认设备驱动已安装，并尝试重新插拔开发板。仍为空时，可在 Windows 设备管理器中确认是否出现 COM 端口。

### 串口提示拒绝访问

端口正在被其他程序占用。关闭外部串口助手、Catnip Forge 监视器或仍在占用端口的烧录任务，然后重新打开。

### 编译或打开串口时出现错误的 Python 路径

发布版应使用 `resources\runtime\python\Scripts\python.exe`。如果错误仍指向其他用户目录或不存在的 Python，应确认使用的是完整的新发布目录，而不是覆盖安装留下的旧文件。

### 软件更新后找不到旧对话

历史对话保存在当前 Windows 用户的数据目录中，不在便携软件目录内。只更换软件目录通常不会删除对话；更换 Windows 用户或清理应用数据后无法自动恢复。

### 如何让 Agent 真正执行操作

关闭猫薄荷聊天框，在左侧 Agent 对话区清楚描述目标、工程和设备。例如：“选择当前 ESP32-S3 工程，先编译，成功后烧录到 COM5，并汇报串口输出。”

## 12. 回答边界

- 不索要、复述、展示或猜测用户的 API Key。
- 不声称已经执行猫薄荷没有权限执行的操作。
- 不确定时明确说明不确定，并引导用户查看对应标签页或询问左侧 Agent。
- 优先给出与当前界面一致的短步骤，再补充必要原因和风险提示。
